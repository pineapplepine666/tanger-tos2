Tang-Er 隱私權政策

明確、透明且維持最小必要數據收集原則

數據收集聲明

本機器人嚴格遵循最小化數據收集原則，僅記錄維持核心功能運作與維護系統安全所需的必要資訊，絕不收集、不保留任何無關隱私。

收集數據項目 (Collected Data)

Discord 使用者名稱與 ID

用於識別發送指令的 Discord 用戶，確保權限系統及個人化功能正常運作。

訊息內容

⚠️ 僅限觸發特定指令時。 機器人平時絕不監聽日常對話，僅在您主動呼叫特定功能時，讀取該指令所需的輸入文字。

伺服器名稱與 ID

記錄機器人所在的 Discord 伺服器識別資訊，用以綁定伺服器專屬設定與防護邏輯。

Discord 元件資料

包含頻道 ID、角色 ID 等必要環境元件資訊，以便機器人判斷執行權限並在正確的位置發送回應。

使用指令時的時間戳記

記錄您觸發指令的精確時間，主要用於計算冷卻時間 (Cooldown)、追蹤異常洗頻，以及後續的錯誤排除分析。

資料用途 (Data Usage)

執行核心功能

所收集之資訊百分之百僅用於系統邏輯運算，確保機器人能針對您的指令做出精確、快速的對應回覆。

系統除錯與優化

當程式發生崩潰或未預期錯誤時，上述數據能協助開發團隊追蹤問題發生當下的環境脈絡，進而修復 Bug 並提升穩定性。

🔒 我們承諾：絕不向任何第三方販售、共享或公開您的任何數據。

資料儲存與刪除 (Storage & Deletion)

儲存安全性

所有收集的最小必要數據皆儲存在具備嚴密防護的加密資料庫中，僅供機器人核心系統調取，防止外部未授權存取。

刪除與申訴

當機器人離開伺服器時，相關配置會定期清理。用戶若需要手動清除個人相關軌跡，可隨時透過聯絡管道向開發團隊提出申請。

政策更新 (Policy Updates)

我們可能會因應 Discord 官方開發者條款變更或機器人功能升級，不定期修改本隱私權政策。重大變更時，我們將會在官方支援管道或透過機器人公告通知用戶。建議您定期查閱本網頁以掌握最新修正內容。
